# 坐标分析与经验教训

## 原始版本坐标分析

原始版本有5台设备，坐标范围：

| 设备 | 纬度 (lat) | 经度 (lng) |
|------|------------|------------|
| 约翰迪尔 S770 收割机 | 46.755 | 131.845 |
| 凯斯 7130 收割机 | 46.760 | 131.880 |
| 约翰迪尔 7M-2204 | 46.745 | 131.850 |
| 凯斯 Puma 2104 (A) | 46.750 | 131.860 |
| 凯斯 Puma 2104 (B) | 46.770 | 131.830 |

### 坐标范围
- **纬度范围**: 46.745 - 46.770 (约 2.8km)
- **经度范围**: 131.830 - 131.880 (约 3.5km)
- **中心点**: 约 46.756°N, 131.853°E

### 关键发现
1. 原始坐标是**手工精确设定**的，都在友谊农场农田内
2. 范围较小（约3km x 3km），确保所有设备都在可控区域
3. fluctuate函数有4个参数，包含min/max限制

## 我犯的错误

1. 扩大坐标范围到 46.72-46.82°N, 131.75-131.87°E（约10km x 10km）
2. 没有验证扩大后的范围是否都是农田
3. 导致部分设备定位到城镇、道路等非农田区域

## 修复方案

基于原始版本的坐标范围，只在已验证的农田区域内生成设备位置：
- 纬度: 46.745 - 46.770
- 经度: 131.830 - 131.880

## 经验教训

1. **不要凭想象设定坐标** - 必须在地图上实际验证
2. **尊重原始设计** - 能工作的代码背后有其道理
3. **小范围比大范围安全** - 宁可设备密集，也不要分布到错误区域
4. **测试验证** - 每次修改后都要在地图上查看效果
